How key permissions work

Apple has no per-key scopes: an API key inherits the role of its API account, set in Apple Business/School Manager under Access Management → Roles. There is no API to query permissions, which is why this page probes them empirically.

To change what a key can do:

  • Broaden or narrow this key — edit the API account's role in ABM/ASM; takes effect on the same key, no re-adding needed.
  • Keep tiered keys — create a second API account with a different role (e.g. a read-mostly one for daily use and a device-manager one for migrations) and add it here as another org profile; switch between them from the header dropdown.

Least-privilege tip: if you mostly browse and report, run day-to-day on the least-privileged role that passes the reads above, and switch to the powerful profile only when you need to assign devices.
